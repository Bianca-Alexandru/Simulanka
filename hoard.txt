# Simulanka Built-in Variables and Functions

## Built-in Variables (Individual Scope)
These variables are available on every individual (e.g., `self.x` or `target.fitness`).

| Variable | Type | Description |
| :--- | :--- | :--- |
| `x` | Int | The X-coordinate of the individual on the matrix. |
| `y` | Int | The Y-coordinate of the individual on the matrix. |
| `fitness` | Int | The calculated fitness score from the last evolution cycle. |
| `species` | String | The name of the species the individual belongs to. |
| `self` | Object | A reference to the individual itself. |

## Built-in Variables (Global Scope)
These variables are available in specific blocks like `EVOLVE` or `RESULT`.

| Variable | Type | Context | Description |
| :--- | :--- | :--- | :--- |
| `generation` | Int | EVOLVE, RESULT | The current generation number. |
| `populationsize` | Int | EVOLVE, RESULT | The number of living individuals. |
| `instance` | Int | RESULT | The ID of the simulation instance (for parallel runs). |
| `avg_fitness` | Int | RESULT | The average fitness of the population. |

## Built-in Functions
Functions can be called within routines or fitness expressions.

| Function | Arguments | Description |
| :--- | :--- | :--- |
| `dist(obj1, obj2)` | Object, Object | Calculates the Euclidean distance between two individuals. |
| `random(min, max)` | Int, Int | Returns a random integer between min (inclusive) and max (exclusive). |
| `len(list)` | List | Returns the number of elements in a list. |

## Lists and Genes
Simulanka supports Lists (Vectors) which are essential for defining complex genes.

- **Literal**: `[1, 2, 3]`
- **Indexing**: `my_list[0]`
- **Assignment**: `my_list[0] = 10;`
- **Length**: `len(my_list)`

## Mutation and Crossover
The `MUTATE` block allows users to define how genes are inherited and modified.

```simulanka
MUTATE {
    crossover: {
        // parent1, parent2, and child are available
        child.genes[0] = parent1.genes[0];
        child.genes[1] = parent2.genes[1];
    }
    mutation: {
        // self (the child) is available
        if (random(0, 100) < 10) {
            self.genes[0] = random(0, 255);
        }
    }
}
```

# Code Flow and Architecture

Simulanka is an interpreted language designed for Genetic Algorithm simulations. The execution follows these stages:

1. **Lexical Analysis (`lexer.rs`)**:
   - The source code is converted into a stream of tokens.
   - Keywords like `ENVIRONMENT`, `SPECIES`, and `FITNESS` are identified.

2. **Parsing (`parser.rs`)**:
   - The token stream is transformed into an Abstract Syntax Tree (AST).
   - The `Program` struct is populated with blocks: `routines_block`, `species_block`, `spawns_block`, etc.

3. **Initialization (`main.rs`)**:
   - The `World` is created based on the `ENVIRONMENT` settings.
   - `World::spawn` creates the initial population, injecting built-in variables (`x`, `y`) into each individual's `Environment`.
   - Species-specific properties (like `genes`) are initialized from the `SPECIES` block.

4. **Simulation Loop (`World::step`)**:
   - For each step (usually 10 per generation):
     - The species-specific `routine` is executed.
     - Routines can use `for target in environment` to sense other individuals.

5. **Evolution (`World::evolve`)**:
   - Fitness is calculated for all individuals using the `FITNESS` block.
   - Individuals are sorted by fitness.
   - The top 50% survive.
   - Reproduction occurs:
     - A `child` is created as a clone of a survivor.
     - If a `crossover` block exists, it combines traits from two parents into the child.
     - If a `mutation` block exists, it randomly modifies the child's traits.
   - A `GenerationSnapshot` is saved for visualization.

6. **Parallel Execution**:
   - Multiple `World` instances run in separate threads if `instances` > 1 is specified in `EVOLVE`.
   - Each thread returns its history to the main thread.

7. **Visualization (`SimApp`)**:
   - An `egui` window opens if `VISUALIZE` is present.
   - Users can navigate through generations using arrows.
   - Modes: `matrix` (shows the best individual's position) or `bars` (shows average fitness growth).
